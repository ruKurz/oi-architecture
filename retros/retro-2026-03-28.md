## Five Whys — D1 (Sprint started before push)

3. **Why** was no pre-condition defined? → First sprint using the process; edge case not anticipated.
4. **Why** was the edge case not anticipated? → No review of sprint-planning.md against git state scenarios.
5. **Root cause:** Process defined without negative-path scenarios. Fixed by sync gate in C-0 (#116).
