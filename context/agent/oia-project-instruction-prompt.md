## Thought Doership — Positioning Principle

*(Based on: John Winsor, "Has AI Ended Thought Leadership?", HBR March 2026)*

**The OIA model gains credibility not through framework completeness, but through operational grounding.**

Thought Leadership is eroding because GenAI produces polished but hollow knowledge at zero cost. Differentiation comes from **Thought Doership**: operational experience, scar tissue from real implementations, willingness to report on failure.

5 warning signs of faux-expertise: no scar tissue · Altitude Lock (30,000 feet, no implementation level) · platitudes about failure · rapidly built expertise · excessively consistent positions.

**Implication for this project:**
- The strongest LinkedIn posts do not show OIA as a framework — they show concrete failure experiences from implementations
- Enterprise Search with real integration problems, failed RAG implementations, data quality in real organizations: this differentiates from generic "Enterprise AI" thought leaders
- Anchor each OIA layer with its own operational scar tissue — that is the difference from a whitepaper

## The OIA Model — Core Concept

**Guiding thought:** *How organizations turn knowledge into action*

**Conceptual flow:**
```
Data → Intelligence → Capability → Solution → Business Outcome
```

**Architecture layers (current state):**

| Layer | Contents |
|---|---|
| Situation Layer | Context: Employee, Event, Time, Place, Domain, Task |
| Use Cases | HR Consulting, Maintenance, Software Dev, Customer Support |
| Solutions & Applications | Enterprise Search, Knowledge Assistants, Code Assist |
| Cognitive Capabilities | Find, Link, Evaluate, Generate, Structure, Report |
| Features & APIs | /search /classify /summarize /link /chat /create |
| AI & Cognitive Infrastructure | LLMs, ML, NLP, Vector Indexes, Knowledge Graphs, RPA |
| Data Layer | Persons, Documents, Projects, Rules + Processing Pipeline |

**Side structures:**
- Left: System Development & Interaction (User Interaction, Context Understanding, Feature Orchestration, Learning & Optimization)
- Right: Data Sources & Processing Pipeline (Ingestion → Processing → Cleaning → Validation)

**Still to be developed (V2 Backlog):**
- Knowledge Core (central concept, not yet fully modeled)
- Actors (Humans & Agents explicitly represented)
- Business Outcome as the closing loop
- Clearer data flow between Pipeline → Knowledge Core → Capabilities
